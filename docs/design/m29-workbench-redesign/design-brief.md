# Milestone 29 多模型管理工作台重设计提案

> 状态：确认用原型，不修改 `web/src`，不代表已经进入实现。

## 设计判断

现有页面的问题不是单个组件尺寸，而是三个层级同时争夺首屏：站点导航、数据库资源树、模型专用工作台都在横向和纵向持续展开。MQ 页面又把概览、消费者、吞吐、消息列表和检查器平铺在同一视口，最终只能依靠 10-12px 字号和窄控件维持布局。

新版采用工具型 IDE 框架：

- 64px 全局导航轨只承载一级模块，Studio 内不重复主站菜单。
- 304px 资源浏览器拥有固定工具栏、独立滚动区和固定状态栏；长树不会挤出视口。
- 数据对象以工作区页签打开，允许同时保留 SQL、MQ topic、关系表等上下文。
- 每个模型使用自己的二级页签。MQ 拆成概览、消息、消费者组、配置，不再一次展示全部信息。
- 详情检查器按需显示在右侧，可收起；结果、历史、维护和高风险操作进入抽屉或明确命令入口。
- 默认正文 14px，树节点 40px，工具栏控件 36px，主标题 20px；不再用微小字号换取容量。

## 借鉴与取舍

| 参考产品 | 借鉴 | SonnetDB 取舍 |
| --- | --- | --- |
| DBeaver | Database Navigator、过滤、Collapse All、Link with editor、对象编辑页签 | 多模型对象共用一棵资源树；增加简单/详细视图，默认不展示冗长 metadata |
| RedisInsight | 数据类型专用 Browser、命名空间浏览、内容格式切换、底部工具入口 | KV 保留专用工作台；全局工具不和资源树混排 |

| MinIO Console | Bucket/Object 渐进浏览、详情面板、治理动作独立 | 对象检查器放右侧，policy/lifecycle/retention 进入专用页签 |
| Kafka UI | Topic → Messages / Consumer Groups / Config 的任务式导航 | MQ 概览、消息、消费者和配置分开；只在概览页展示完整监控 |

| 资源浏览器 | 304px | 可收起，可拖拽调整，树区独立滚动 |

## 深浅主题

深色与浅色模式共用完全相同的框架尺寸、信息架构和交互位置，不允许因主题切换造成布局跳动。

### 深色模式

以 `concept-mq.png` 为确认基准：深色顶部栏和功能轨承载全局导航，资源浏览器、工作区和检查器使用高对比浅色表面；青绿色用于 Studio 激活状态和主要选中状态。

### Microsoft 365 浅色模式

以 `concept-framework-microsoft365-light.png` 为确认基准：

- 外层画布和框架 chrome 使用近白基底，并叠加低饱和冰蓝、薄荷绿、蜜桃和淡紫珠光环境色。
- 资源树、数据表和检查器保持白色或近白高对比表面，柔和彩色不得进入大面积数据单元格。
- `#0F6CBD` 用于 Fluent 蓝交互状态，`#061F2C` 用于主要操作，`#512BD4` 只用于少量 Studio/.NET 识别。
- 健康、告警和危险状态分别使用 `#107C10`、`#F7630C`、`#C50F1F`，不依赖主题背景表达状态。
- Copilot 多色只允许出现在 AI 入口等小尺寸识别元素，不作为页面主背景。

## 原型中的关键行为

- 点击资源树中的 `telemetry.raw` 或 `orders` 可切换 MQ / 关系表工作台。
- 顶部工作区页签可在 SQL、MQ 和关系表之间切换。
- 资源浏览器可以整体收起，数据库分组可以展开或折叠。
- 搜索框会过滤树中的对象名称和类型。
- MQ 的二级页签只呈现当前任务；消息详情检查器可以收起。
- 两个工作台沿用同一外壳，证明框架不是只为 MQ 页面定制。

## 实现边界建议

确认原型后再拆成实现任务。第一批只改共享外壳、Explorer、字号/密度 token 和 MQ/关系表的布局挂载，不改 API、权限、写审批、历史和模型语义；其他六个工作台按共享框架逐个迁移，避免一次性重写。
